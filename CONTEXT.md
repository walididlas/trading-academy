## 1. What This Is

A personal, full-stack trading education platform built for one user. It teaches the ICT + ICC Kill Zone methodology from zero to execution, connects live to TradingView Desktop, scans markets in real time, and acts as an always-on AI trading assistant.

**Not a SaaS. Not multi-user. Built for one person ‚Äî the owner.**

## 6. The Trading Methodology

### Pairs Traded
| Pair | Role |
|---|---|
| XAUUSD | Primary (high-volatility, USD-driven) |
| EURUSD | Primary |
| GBPUSD | Primary |
| NZDJPY | Secondary |

Scanner also supports: GBPJPY, USDJPY, AUDUSD, USDCAD, USDCHF

### Timeframes
- **H1** ‚Äî signal detection and EMA50 trend filter
- **M15** ‚Äî Order Block and FVG refinement  
- **M5** ‚Äî precise execution

### Kill Zones (UTC = Morocco ‚àí1h)
| Zone | UTC | Morocco | Notes |
|---|---|---|---|
| London KZ | 09:00‚Äì12:00 UTC | 10:00‚Äì13:00 | Primary. Best for all pairs. |
| NY KZ | 14:30‚Äì17:30 UTC | 15:30‚Äì18:30 | Strong setups, especially post-London continuation. |

**Rule: No trades outside Kill Zones. Ever.**

### The ICC Method (Indication / Correction / Continuation)

All 5 conditions must be met:

```
1. KILL ZONE TIMING
   Trade must be during London or NY KZ.

2. TREND ALIGNMENT (EMA50 on H1)
   Long trade ‚Üí price ABOVE 50 EMA on H1
   Short trade ‚Üí price BELOW 50 EMA on H1

3. BASE CANDLE (The "Indication")
   Body/range ratio ‚â• 60%. Trend-aligned. Breaking recent swing.
   This is the institutional displacement candle.

4. CORRECTION (Pullback)
   Price retraces ‚â• 30% back into the base candle zone.
   Wait for this ‚Äî never enter on the base candle itself.

5. CONTINUATION (Entry)
   Price closes beyond base candle high (long) or low (short).
   Entry: at the continuation close price.
   Stop Loss: below/above base candle + 2-pip buffer.
   Take Profit: 3:1 R:R minimum ‚Üí next BSL/SSL pool.
```

### The Signal Scoring System (0‚Äì100 points) ‚Äî v2
| Factor | Points | Notes |
|---|---|---|
| Kill Zone active | +25 | London 09-12 UTC / NY 14:30-17:30 UTC |
| Valid Order Block (OB) | +20 | Prefers M15; H1+M15 confluence noted |
| Fair Value Gap (FVG) | +15 | Pair-correct pip size (fixed v1 bug) |
| Market Structure (BoS or CHoCH) | +15 | From swing high/low analysis |
| EMA50 trend aligned | +10 | Always scores (sets direction) |
| Premium / Discount zone | +10 | Longs in discount (<50%), shorts in premium (>50%) |
| No recent high-impact news | +5 | 20-min window |

**Grade thresholds:** STRONG ‚â•80 ¬∑ WATCH 60‚Äì79 ¬∑ MONITORING <60

**Signal also includes:** `confluence_count` (0-7), `premium_discount` (zone, pct, correct), `atr`, `liq_targets` (BSL/SSL swing levels), `rr1/rr2/rr3` (actual R:R from swing targets)

### ICT Concepts Used
| Concept | Usage |
|---|---|
| BSL / SSL | TP targets ‚Äî price delivers to opposing liquidity |
| IPDA 20-day | Macro TP targeting (20-day high/low as next delivery zone) |
| Order Blocks | Last opposing candle before impulse ‚Üí institutional zone |
| Fair Value Gaps (FVG) | Price imbalance ‚Üí return-to-fill target |
| Premium / Discount | Enter discount for longs, premium for shorts |
| Market Structure (HH/HL, LH/LL) | Trend confirmation + bias |
| BoS / CHoCH | Trend continuation vs warning of reversal |

### Risk Rules
| Rule | Value |
|---|---|
| Max risk per trade | 1‚Äì2% of account |
| Minimum R:R | 3:1 |
| Target R:R | 5:1 |
| Daily loss limit | Stop after 2 consecutive losses |
| Weekly drawdown limit | Stop after 3% |

## 8. The Signal Scanner

**File:** `backend/scanner.py`

- Runs every **2 minutes** as a background `asyncio` task
- Pairs: XAUUSD, EURUSD, GBPUSD, NZDJPY
- Timeframes: H1 (trend + OB) + M15 (refined OB + FVG)

### Scoring Logic
1. Kill Zone check (UTC times)
2. EMA50 calculated from last 50 H1 closes
3. Order Block: last opposing candle before impulse, price inside zone
4. FVG: 3-candle gap ‚â• 3 pips, price at/inside
5. Market Structure: swing high/low analysis with BoS/CHoCH detection
6. News check: `get_cached_news()` from news_fetcher ‚Äî deducts 10pts if HIGH news in last 30min

### Signal Format
```python
{
  pair, direction, timeframe, score, grade,
  criteria: { kill_zone, order_block, fvg, market_structure, ema50, news_clear },
  entry, sl, tp1, tp2, tp3,
  pip_sl, pip_tp1, pip_tp2, pip_tp3,
  ob_zone, fvg_zone, ema50,
  timestamp, expires_at  # 4 hours from detection
}
```

### OHLCV Data Flow
```
MCP tools ‚Üí POST /api/feed ‚Üí _ohlcv_cache["{SYMBOL}_{TF}"]
                                        ‚Üì
                               scanner._get_ohlcv(pair, tf)
                                        ‚Üì
                          CDP direct fetch (fallback)
```

## 9. Real-Time News System

**File:** `backend/news_fetcher.py`

- Sources: ForexLive (`/feed/`) + FXStreet (`/rss/news`)
- Polled every **60 seconds** via `httpx`
- Impact classification: HIGH (30+ keywords: cpi, nfp, fed, fomc, powell, gdp...) / MEDIUM / LOW
- Currency detection: `currencies` field is a list, e.g. `["USD", "GBP"]`
- Deduplication by MD5 hash of `source:title`
- Broadcasts `{ news_update: [...] }` via WebSocket on new items

**File:** `backend/calendar_fetcher.py`

- Source: ForexFactory JSON API (this week + next week)
- Refreshes every **30 minutes**
- `get_news_risk_for_pair(pair, window_minutes=30)` ‚Üí `{ level, events, next_event }`
  - Checks upcoming AND past 5 min events
- `get_next_high_event()` ‚Üí next global HIGH event with `minutes_until`
- `get_cached_calendar()` ‚Üí full dict with `upcoming_30m`, `upcoming_60m`, `next_high`, `today`, `risk_by_pair`

**Endpoint:** `GET /api/calendar`

## 11. Alert & Notification System

**File:** `src/contexts/AlertContext.jsx`

Single WebSocket connection at `ws://hostname:8000/ws/signals`. Shared across all pages.

### Toast Types
| Type | Sound | Duration | Trigger |
|---|---|---|---|
| `signal` | 3 beeps | 12s | STRONG signal detected |
| `killzone` | ascending chime | 7s | Kill Zone opens |
| `confluence` | 3 beeps | 7s | HIGH news + Kill Zone simultaneously |
| `warning` | 3 beeps | 7s | HIGH news + open journal trades OR STRONG signal |

### Currency Matching (AlertContext)
When HIGH news drops via `news_update`, the system:
1. Checks `PAIR_CURRENCIES` map to find which pairs are affected
2. Compares against current STRONG signals (`signalsRef.current`)
3. Fires: "‚õî News Risk ‚Äî {pair} STRONG signal" toast + native notification
4. Checks open journal trades (`trading_journal` localStorage) for pair match
5. Fires: "‚öÝ High news ‚Äî X open trades at risk" with pair names

### NewsTicker (Dashboard + Signals)
- Scrolls top 5 news headlines
- `nextHighEvent` prop ‚Üí pinned countdown: "‚ö° NFP in 14m"
- `news-ticker-pulse-gold` class: slow gold glow when KZ active + HIGH news in feed
- `news-ticker-flash-red` class: red flicker 3s when new HIGH news arrives

## 15. The Trade Journal

**File:** `frontend/src/pages/Journal.jsx`

- `localStorage` key: `trading_journal`
- All calculation is **client-side only** ‚Äî no backend required

### Pip & P&L Calculation
```js
PIP_SIZE  = { XAUUSD: 0.1, GBPJPY: 0.01, USDJPY: 0.01, NZDJPY: 0.01, default: 0.0001 }
PIP_VALUE = { XAUUSD: 100, GBPJPY: 6.67, NZDJPY: 6.67, default: 10 }  // per standard lot

pips = (exit - entry) / pipSize  √ó direction_sign
pnl  = pips √ó pipValue √ó lotSize
rr   = reward_pips / risk_pips
```

### ICC Grading (A/B/C/F)
4 criteria, 1 point each:
1. Kill Zone (checkbox)
2. Trend Aligned (checkbox)
3. ICC Valid (checkbox)
4. R:R ‚â• 3 (auto-computed from entry/sl/tp)

Score ‚Üí A=4, B=3, C=2, F‚â§1

### Trade Object Schema
```js
{
  id, date, pair, direction, session,       // meta
  entry, sl, tp, exitPrice, lotSize,        // prices
  pips, pnl,                                // auto-computed
  rr, grade,                                // auto-computed
  kz, trendAligned, iccValid,               // ICC checkboxes
  result, notes, status,                    // outcome ('open'|'win'|'loss'|'be')
  tp1, tp2, signalScore, signalGrade        // from signal card (Paper Trade)
}
```

### UI Features
- **Equity Curve** ‚Äî SVG cumulative pips chart (polyline + area fill, zero baseline, trade dots)
- **TradeForm** ‚Äî real-time auto-compute of pips/pnl/rr/grade as user types; ICC checkboxes; grade preview badge
- **TradeCard** ‚Äî expandable card with grade ring (A/B/C/F color-coded), session tag, pips + dollar display, ICC checklist in expanded view
- **WeekGroup** ‚Äî weekly header with W/L count, total pips + P&L for that week
- **GradeBar** ‚Äî horizontal A/B/C/F distribution bar
- **Open Positions** ‚Äî shown at top of page with pulsing green dot, separate from closed trades
- **Stats** ‚Äî Win Rate, Total P&L ($), Profit Factor, Expectancy (pips/trade), avg R:R, grade distribution
- **Filters** ‚Äî pair filter (dynamic from actual trades) + ALL/WIN/LOSS/BREAKEVEN result tabs
